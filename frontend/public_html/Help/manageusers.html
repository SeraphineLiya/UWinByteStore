Help & User Guide: Managing User Accounts

User accounts can be managed easily through the front-end admin portal. Upon logging in and accessing the appropriate page, you will be presented with a list of all user accounts currently in the website's database. This list includes the user account ID (automatically generated), along with the email address / username of the account, their first and last name, as well as the current status of their account.

Currently valid user account statuses are 'Pending', 'Blocked', and 'Admin'. Pending status only applies to user accounts that have not yet been assigned a permission level by an admin. The account list will be sorted so that such accounts are always displayed at the top. Blocked status simply means that admin permissions were not granted to the account, while Admin status means that they were.

To change an account's status, simply select the checkbox that applies to the accounts whose status you want to change, then click the button to either 'Make Admin' (set status to Admin), 'Block Admin' (set status to Block), or 'Delete' to delete that user's account permanently. Be very careful with these options, as you can change or delete your own account here too. If all admin accounts are deleted, the system admin must manually set the permission level of an account to 'Admin' again to regain access to the front-end interface, following the same procedure as when the website is created.
